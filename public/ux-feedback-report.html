AU 0101 App UX Feedback Report

User Experience Analysis and Recommendations

The app name "AU 0101" may create some confusion; it appears like a beta or test version.

The initial animation sequence with three slides feels longer than necessary; the "Proceed" button adds limited value.

The SMS Secure Code feature opens the mobile SMS app and requires disabling Wi-Fi to use mobile data. It also asks for permission to send SMS and takes noticeable time (around 5 seconds) to verify. This can be taken up later in the login flow.

There's a ~30-second wait time during mobile number verification without any visible loader or progress indicator.

On the home page, when selecting "Open Savings," the mobile number is pre-filled, so the placeholder for the Aadhaar-linked number isn't shown. However, it reappears when PAN or Aadhaar verification fails.

During PAN entry, the keyboard collapses after typing alphabets, requiring the user to tap again to continue entering numbers and alphabets alternately.

OTP is not getting auto-populated after it's received.

The landing page becomes available only after completing multiple steps; it would improve usability if accessible earlier.
